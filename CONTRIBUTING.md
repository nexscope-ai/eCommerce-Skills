# Contributing to E-Commerce Skills

Thanks for your interest in contributing! We welcome high-quality skills that help e-commerce businesses grow.

## 🎯 What Makes a Good Skill

### ✅ We Want
- **Actionable output** — Results that users can immediately implement
- **Clear workflow** — Defined input → process → structured output
- **Cross-platform** — Works for Shopify, WooCommerce, Etsy, eBay, etc. (not just one platform)
- **Business value** — Solves real problems that e-commerce sellers face
- **Beginner friendly** — Non-technical users should be able to follow along

### ❌ We Don't Want
- Information-only skills (use Google instead)
- Skills requiring API keys that most users don't have
- Platform-specific skills that need admin access
- Overly complex multi-step workflows
- Skills that duplicate existing functionality

## 📋 Skill Requirements

### 1. Structure
```
your-skill-name/
├── SKILL.md           # Required: Main skill file
├── examples/          # Optional: Usage examples
├── scripts/           # Optional: Helper scripts
└── README.md          # Optional: Additional documentation
```

### 2. SKILL.md Format
Your `SKILL.md` must include:

```markdown
# Skill Name

Brief description of what this skill does and the business value.

## What it does

**Input:** Clear description of what data/information is needed
**Output:** Specific description of what the user gets
**Use case:** When would someone use this?

## Instructions

Step-by-step process...

## Example

Concrete example with real inputs and expected outputs...

## Limitations

What this skill can't do, important caveats, etc.
```

### 3. Naming Convention
- Use kebab-case: `ecommerce-email-sequences`
- Start with platform if platform-specific: `shopify-conversion-audit`  
- Use generic names for cross-platform skills: `product-launch-checklist`

## 🚀 Contribution Process

1. **Fork** this repository
2. **Create** a new branch: `git checkout -b skill/your-skill-name`
3. **Add** your skill in a new directory
4. **Test** your skill with real examples
5. **Submit** a pull request with:
   - Description of the skill's business value
   - Example of input/output
   - Target audience (beginners/advanced/etc.)

## 📊 Skill Quality Standards

We evaluate contributions based on:

- **Clarity**: Can a non-technical user follow the instructions?
- **Completeness**: Does it deliver the promised output?
- **Practicality**: Is the output immediately actionable?
- **Uniqueness**: Does it add new value vs. existing skills?

## 🤝 Getting Help

- **Questions**: Open an issue with the `question` label
- **Ideas**: Open an issue with the `enhancement` label  
- **Bugs**: Open an issue with the `bug` label
